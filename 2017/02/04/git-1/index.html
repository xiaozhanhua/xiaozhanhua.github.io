将本地项目上传至GitHub

Feb 4, 2017

GitHub 是一个面向开源及私有软件项目的托管平台，我们不但可以在上面学习众多优秀的开源框架、项目demo，还可以把自己的项目代码放上去。今天我们就来学习怎么通过git客户端将自己本地的项目上传至GitHub。

安装客户端

下图便是git客户端安装的启动页，安装过程比较简单。
可参照这篇博文：GitHub超详细图文攻略。

下面要说的是如何通过git客户端将自己本地的文件上传至GitHub。

具体步骤

（1）在本地电脑上创建.git仓库

在自己的本地电脑任意位置，右键选择 Git Gui

之后会弹出以下框，我们选择创建新的版本库

然后会出现让你输入一个文件目录名的弹窗，我们直接输入一个文件名就行。这里因为我要存放的是2048的游戏项目，所以我输入的是2048

完成点击新建以后，会出现一个可视化窗口，与此同时我们会发现本地电脑多了一个名为2048的文件夹，打开该文件夹会发现里面还有一个隐藏的 .git 文件夹。这时，这个2048文件夹就是你的项目仓库，只要将你需要上传的项目放在该文件夹即可。这里我放入的是命名为 TheGame2048 的项目文件。

（2）在GitHub网站上创建版本库

在自己的GitHub主页上，点击New repository

之后会跳转到 Create a new repository 页面，要我们为当前版本库输入一个项目名称。在这里我输入的是 2048，然后点击 Create repository 即可。

最后我们把当前版本库的链接复制一下，因为后面会用到。我这里的链接为：https://github.com/xiaozhanhua/2048.git，现在我们就可以把本地文件上传到Github上了。

（3）将本地文件上传至Github

在步骤1后面出现的可视化窗口上，我们依次选择 远端(remote) —— Add

会出现一个弹框，在第一栏输入我们的Github用户名，第二栏粘贴刚才拷贝下来的链接地址，点击Add，显示绿色背景成功代表创建成功

接下来我们从上到下，依次选择 重新扫描——缓存改动——签名——提交——上传这几个步骤即可。需要注意的是，在上传步骤，要在“传输选项”选择其中一个，否则无法上传成功。

点击上传按钮之后，会依次出现两个弹框，分别要你输入GitHub用户名及密码

（4）文件上传成功

输入用户名及密码后，隔一会儿出现以下弹框则表示上传成功

这时到自己的GitHub上去看，会发现本地项目文件已经上传进来了

至此， 通过git客户端将本地项目上传至GitHub 已经演示完毕。
